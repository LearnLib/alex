# Frequently asked questions

Can I test and learn any web application that is accessible over the internet?
Yes, but since learning usually takes a lot of traffic, assure you are allowed to make a lot of requests to the target application. The primary use case is to use ALEX for testing applications that are installed locally or in the same network.

Do I really not have to have any programming experience?
We made best efforts to abstract all necessary steps to test a web application with ease. While modeling tests for your application, it may be helpful to have basic understanding of HTML and CSS. The same applies for modelling REST tests where it is required that you have basic understanding of the JSON format and the HTTP protocol.

What are the specifications for running ALEX?
Learning is a CPU and memory intense process, but we cannot give a lower bound for your system specs. From our own observations, it is advised to have at least 4Gb of RAM and a relatively modern Dual Core CPU to run tests at a satisfiable performance. The more power your system has, the faster your tests should (ideally) execute.

I want to learn a specific feature of my application but ALEX does not provide a way to model it.
We do not develop ALEX as a solution to all situations that may occur in a web application, but to the most common ones. If you think a key feature is missing, feel free to submit an issue or create a pull request at GitHub and we will review it.
